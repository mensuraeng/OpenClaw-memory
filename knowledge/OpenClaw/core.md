# core.md

## Escopo
Produto principal e núcleo técnico do ecossistema.

## Repositório principal
- `openclaw/openclaw` — core do produto.

## Quando consultar este arquivo
- arquitetura do OpenClaw
- gateway
- runtime
- tools
- skills como conceito de produto
- comportamento do agente
- sessões, canais, cron, nodes, memória e operação central

## Leitura correta
Não tratar este repo como sinônimo de todo o ecossistema. Ele é o núcleo, não o mapa completo.

## Relações mais importantes
- documentação oficial vive em `openclaw/docs`
- ecossistema de skills vive em `openclaw/clawhub` e `openclaw/skills`
- operação avançada expande o core via `openclaw/lobster` e `openclaw/acpx`

## Regra prática
Se a dúvida for “como o OpenClaw funciona”, começar aqui e depois cruzar com `docs.md`.
