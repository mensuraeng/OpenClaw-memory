# Checklist de retomada operacional

_Atualizado em 2026-04-21 13:59 BRT_

## Regra
Antes de responder ou agir em qualquer frente crítica, a retomada deve passar por este checklist.

## Etapas obrigatórias
1. identificar a frente correta
2. consultar a memória temática da frente
3. localizar o estado operacional atual
4. distinguir explicitamente:
   - planejado
   - preparado
   - executado
   - bloqueado
5. identificar:
   - padrão atual
   - fallback
   - nível de productização
6. checar o último teste real ou evidência validada
7. checar o bloqueio real atual, se houver
8. identificar o próximo passo correto
9. só então responder, agir ou escalar

## Regra de resposta
Se a frente estiver apenas preparada, nunca responder como se estivesse executada.

## Regra de bloqueio
Se houver bloqueio real, responder com:
- causa real
- evidência do bloqueio
- único input necessário para destravar, se existir

## Regra de memória
Se uma capacidade foi validada ou corrigida no ciclo atual, ela deve subir para memória durável no mesmo ciclo.
