# Checklist Pré-Envio — Redator Executivo de Obra

## 0. Verificação rápida (responder antes de entregar)

- [ ] O objetivo do texto está claro (informar / cobrar / notificar / solicitar / registrar)?
- [ ] O destinatário foi identificado ou placeholder declarado?
- [ ] O nível de tensão foi classificado (Baixo / Médio / Alto / Crítico)?
- [ ] A formalidade está compatível com o destinatário e a situação?
- [ ] A primeira frase diz o assunto — sem cumprimento, sem preâmbulo?
- [ ] Nenhum clichê proibido foi usado?
- [ ] O texto termina com ação esperada, responsável e prazo?
- [ ] O nível de confiança foi declarado?

Se qualquer item for **não** → corrigir antes de entregar.

---

## 1. Checklist por tipo de documento

### E-mail executivo
- [ ] Assunto no formato: [OBRA] — [Assunto principal] — [Data ou referência]
- [ ] Primeira frase = assunto do e-mail, não saudação
- [ ] Máximo 3 parágrafos no corpo
- [ ] Parágrafos com máximo 4 linhas
- [ ] Negrito usado no máximo 2 vezes (prazo ou valor crítico)
- [ ] Fechamento sem clichê
- [ ] Encaminhamento claro: ação + responsável + prazo

### E-mail de cobrança
- [ ] "Pendência" ou "prazo" aparecem no primeiro parágrafo?
- [ ] Valor da pendência declarado?
- [ ] Prazo para regularização declarado?
- [ ] Consequência do não pagamento / não cumprimento declarada?
- [ ] Tom firme e factual — sem adjetivos emocionais?
- [ ] Prazo indicado ainda é válido (não vencido)?

### Ata de reunião
- [ ] Tema / título declarado?
- [ ] Data e horário declarados?
- [ ] Participantes listados (ou "participantes não informados" sinalizado)?
- [ ] Pontos discutidos em ordem cronológica?
- [ ] Deliberações separadas dos pontos de discussão?
- [ ] To-dos com: ação + responsável + prazo (cada um)?
- [ ] Próxima reunião ou prazo de revisão declarado?

### Comunicado de atraso
- [ ] Referência contratual (nº do contrato ou cláusula)?
- [ ] Causa do atraso declarada de forma objetiva?
- [ ] Impacto em dias declarado?
- [ ] Ação de recuperação proposta?
- [ ] Novo prazo previsto?
- [ ] Tom compatível com o nível de tensão?

### Notificação de não conformidade
- [ ] Descrição objetiva do fato (sem adjetivos)?
- [ ] Data de ocorrência declarada?
- [ ] Base contratual citada (cláusula, item, norma)?
- [ ] Exigência específica declarada?
- [ ] Prazo de cura declarado?
- [ ] Nota de revisão jurídica incluída se nível Crítico?

### Comunicado de paralisação
- [ ] Causa da paralisação declarada?
- [ ] Data de início declarada?
- [ ] Impacto no cronograma (dias) declarado?
- [ ] Condição para retomada declarada?
- [ ] Responsável pela resolução identificado?
- [ ] Nota de revisão jurídica incluída se responsabilidade contratual?

### Mensagem WhatsApp
- [ ] Sem saudação formal?
- [ ] Parágrafos com máximo 3 linhas?
- [ ] Negrito em asterisco (*texto*) para datas e valores críticos?
- [ ] Máximo 2 emojis operacionais (🔴 ✅ ⚠️)?
- [ ] Sem assinatura corporativa extensa?
- [ ] Versão separada da versão e-mail?

---

## 2. Checklist de consistência

- [ ] Nenhum prazo vencido sem marcação explícita?
- [ ] Nenhuma informação contraditória sem declaração?
- [ ] Tom solicitado compatível com a situação real?
- [ ] Placeholders sinalizados claramente para o usuário preencher?
- [ ] Premissas assumidas declaradas nas notas de uso?

---

## 3. Checklist de despacho via OpenClaw

- [ ] Canal de envio definido (e-mail Graph API / WhatsApp / documento)?
- [ ] Confirmação explícita do Alê obtida antes de enviar?
- [ ] Versão interna vs. externa diferenciada quando necessário?
- [ ] Informação sensível removida da versão que vai ao cliente?

---

## 4. Critério de aceite final

A saída só está completa quando:
- [ ] Nível de confiança declarado
- [ ] Nenhum clichê proibido presente
- [ ] Texto termina com próximo passo claro
- [ ] Placeholders sinalizados
- [ ] Canal de envio indicado
- [ ] Nota jurídica incluída se nível Crítico
