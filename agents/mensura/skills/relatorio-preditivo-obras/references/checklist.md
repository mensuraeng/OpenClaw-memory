# Checklist — Relatório Preditivo de Obras

> Verificar cada item antes de entregar qualquer relatório. Um item marcado ❌ = declarar explicitamente na seção "Limitações" do relatório.

---

## CHECKLIST DE ENTRADA

- [ ] Data-base do cronograma identificada e declarada
- [ ] Tipo de relatório definido: Completo (1ª análise) / Comparativo semanal / Resumo executivo
- [ ] Prazo contratual e baseline original registrados
- [ ] Informação de multa por atraso e custo indireto mensal disponíveis (ou ausência declarada)
- [ ] Se comparativo: relatório anterior acessado e deltas calculados
- [ ] Campos presentes e ausentes inventariados (Passo 0 executado)

---

## CHECKLIST DE CONTEÚDO

### Indicadores EVM
- [ ] SPI calculado (ou justificativa de impossibilidade)
- [ ] BEI calculado (ou declarado como indisponível)
- [ ] CPLI calculado para atividades do caminho crítico
- [ ] EAC de prazo calculado com os 3 cenários
- [ ] Origem de cada número rastreável (não inferida sem base)

### Caminho crítico
- [ ] Identificado ou impossibilidade justificada (sem predecessoras = declarar)
- [ ] Folga total calculada para atividades críticas
- [ ] Impacto de atrasos no caminho crítico declarado em dias

### Diagnóstico
- [ ] Semáforo definido com critério explícito (não apenas visual)
- [ ] Top 3 riscos identificados com causa-raiz (não lista de atividades)
- [ ] Padrões sistêmicos identificados (mesmo problema repetido = solução estrutural)

### Plano de ação
- [ ] Toda ação tem responsável definido (Obra / Cliente / Projetista / Fornecedor)
- [ ] Toda ação tem prazo específico
- [ ] Toda ação é acionável (não "monitorar" ou "acompanhar")
- [ ] Ações da semana anterior auditadas: executadas / não executadas / em andamento

### Se comparativo semanal
- [ ] Deltas calculados: SPI semana x semana
- [ ] Ações não executadas escaladas (2ª semana: alerta / 3ª semana: escalar)
- [ ] Tendência de avanço declarada (aceleração / desaceleração / estabilidade)

---

## CHECKLIST DE ENTREGA

- [ ] Nível de confiança da análise declarado (Alto/Médio/Baixo + justificativa)
- [ ] Linguagem direta: "vai atrasar 31 dias" ✅ | "possível variação temporal" ❌
- [ ] Nenhuma premissa otimista não suportada pelos dados
- [ ] Limitações declaradas quando dados ausentes ou inconsistentes
- [ ] Se pipeline automático: JSON de KPIs salvo em memória
- [ ] Se OpenClaw: notificação WhatsApp gerada para Alexandre (Resumo Executivo — Seção C)
