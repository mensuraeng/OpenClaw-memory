
## Mission Control — correção de cache/UI antiga

- Alê relatou que ao entrar no Mission Control aparecia sempre a versão antiga, como se a UI nova não subisse.
- Causa encontrada: `projects/mission-control/public/sw.js` usava service worker cache-first (`mc-v1`) para páginas não-API, podendo servir app shell antigo.
- Correção aplicada: service worker trocado para limpar caches no `activate`, usar `no-store` para navegação/documentos/API e não cachear app shell.
- `next.config.mjs` atualizado com headers anti-cache globais: `Cache-Control: no-store, no-cache, must-revalidate, proxy-revalidate`, `Pragma: no-cache`, `Expires: 0`.
- Validação: `npm run build` OK; PM2 `mission-control` reiniciado; `https://mc.mensuraengenharia.com.br/openclaw` retorna `HTTP 307` para login com headers anti-cache; `https://mc.mensuraengenharia.com.br/sw.js` retorna `HTTP 200`, `cf-cache-status: BYPASS` e novo tamanho `1022` bytes.
