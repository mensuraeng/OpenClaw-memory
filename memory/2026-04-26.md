- Conclusão consolidada: Scalp pode ser adotado apenas como **Strategy Lab — High Risk / Intraday Research**, com etiqueta `SCALP_RESEARCH_ONLY`.
- Bloqueado para agora: dinheiro real, sinal para execução manual, ordem automática, conexão com corretora, alavancagem/margem/short/opções/cripto e qualquer uso de backtest isolado como prova de edge.
- Motivo: scalp é day trade intradiário sensível a spread, taxas, slippage, latência, execução parcial, fila no book, custos de dados e overfitting.
- Evidências usadas: estudos de Barber/Odean sobre day traders em Taiwan; paper “The cross-section of speculator skill”; estudo FGV/RePEc “Day trading for a living?” sobre day trade no Brasil; referências educacionais como Investopedia; repositórios GitHub de scalping bots/backtests e frameworks como `backtesting.py`.
- Evidência forte registrada: no estudo brasileiro, cerca de 97% dos traders persistentes perderam dinheiro; parcela minúscula supera retornos relevantes com consistência.
- Repositórios de scalp encontrados servem como referência didática, mas em geral são frágeis para execução real por ignorarem microestrutura, custo realista, slippage, latência e robustez fora da janela otimizada.
- Requisito mínimo antes de qualquer Paper/Shadow de scalp: dados intraday adequados, modelo explícito de custo/slippage, backtest sem lookahead, out-of-sample, walk-forward, múltiplos regimes, mínimo 30–60 pregões de forward test sem ordem, Risk Gate e journal obrigatório.

## Decisão operacional — interfaces agent-native / CLI Anything
- Alexandre enviou transcrição de vídeo sobre tese “CLI anything”: agentes ficam limitados quando software só tem GUI; o ganho real vem de interfaces estruturadas, CLI, JSON, testes e skills/rails.
- Avaliação consolidada: a tese agrega diretamente à realidade Mensura/OpenClaw, mas deve ser aplicada com foco, não como conversão indiscriminada de qualquer software.
- Prioridade aprovada: criar CLIs internas para sistemas que já têm dor operacional e dados reais. Ordem recomendada: primeiro `mensura-schedule`; depois `trade`.
- Diretriz arquitetural adotada: `scripts soltos → CLI interna → skill/agent map → cron/workflow → dashboard`, sempre com logs, idempotência, output estruturado e guardrails.
- Trade pode ter CLI para radar, gold, notícias, carteira e risco, mas segue proibido para execução real de ordens/aplicações/resgates sem autorização humana explícita.
