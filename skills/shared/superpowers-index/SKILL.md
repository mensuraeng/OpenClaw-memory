# Superpowers Skills — Índice

Skills avançadas de desenvolvimento instaladas em `/root/.openclaw/workspace/skills/`.

## Skills disponíveis

| Skill | Quando usar |
|-------|-------------|
| `brainstorming` | Antes de qualquer implementação — explorar requisitos e design |
| `writing-plans` | Criar planos de implementação detalhados |
| `executing-plans` | Executar plano step-by-step com checkpoints |
| `dispatching-parallel-agents` | Dividir tarefas em sub-agentes paralelos |
| `subagent-driven-development` | Desenvolvimento orientado a sub-agentes |
| `systematic-debugging` | Debug estruturado e sistemático de problemas |
| `test-driven-development` | TDD: testes antes da implementação |
| `requesting-code-review` | Solicitar revisão de código |
| `receiving-code-review` | Processar e aplicar feedback de code review |
| `finishing-a-development-branch` | Finalizar branch antes do merge |
| `using-git-worktrees` | Usar git worktrees para trabalho paralelo |
| `verification-before-completion` | Verificar qualidade antes de declarar done |
| `writing-skills` | Criar novas skills para o sistema |
| `using-superpowers` | Meta-skill: como encontrar e usar as outras skills |

## Regra de ouro

**Se há 1% de chance que uma skill se aplica, invoque-a.**

Superpowers skills sobrescrevem comportamento padrão:
1. Instruções explícitas do usuário (CLAUDE.md, AGENTS.md) — maior prioridade
2. Superpowers skills
3. Comportamento padrão do sistema

para `/root/.openclaw/workspace/skills/` — cada skill em seu próprio diretório com SKILL.md.
